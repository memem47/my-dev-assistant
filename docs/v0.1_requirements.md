# v0.1 Requirements: Minimum CLI Prototype

## 1. Purpose

v0.1 defines the minimum command-line version of My Dev Assistant.

The purpose of this version is to confirm that a short task description can be converted into basic development artifacts.

v0.1 is not intended to be a complete automation tool.
It is intended to provide a small testable, and practical starting point.

---

## 2. Scope

v0.1 includes:

- Accepting a short task description from the command line
- Generating an Issue title
- Generating an Issue description
- Generating a branch name
- Generating a Conventional Commit message
- Generating a Pull Request title
- Generating a Pull Request description
- Printing the generated result to the console

v0.1 does not include:

- GUI
- GitHub API integration
- OpenAI API integration
- Git command execution
- Database or history saving
- Configuration files
- Automatic copy to clipboard

---

## 3. Input Requirements

The tool shall accept one task description as input.

Example:

```bash
python -m my_dev_assistant "Add error handling for invalid RAW image size"
```

Input rules:

- The input shall be a non-empty string
- The input shall describe one development task
- If the input is empty, the tool shall show an error message

---

## 4. Output Requirements

The tool shall output the following items:

- Issue title
- Issue description
- Branch name
- Commit message
- Pull Request title
- Pull Request description

Example output:

```txt
Issue title:
Invalid RAW image size error

Issue description:
Add error handling for invalid RAW image size.

Branch:
fix/invalid-raw-image size

Commit:
fix: handle invalid RAW image size

PR title:
Fix invalid RAW image size handling

PR description:
## Summary

This PR adds error handling for invalid RAW image size.

## Changes

- Add validation for RAW image size
- Return an error when the RAW image size is invalid
```

## 5. Generation Rules

v0.1 uses simple rule-based generation.

### 5.1 Branch Name

The branch name shall follow this format:

```
<type>/<short-description>
```

Allowed types:

- feat
- fix
- docs
- test
- refactor
- chore

If the type cannot be determined, chore may be used.

Example:

```txt
fix/invalid-raw-image-size
```

### 5.2 Commit Message

The commit message shall follow the Conventional Commits style:

```txt
<type>: <summary>
```

Example:

```txt
fix: handle invalid RAW image size
```

### 5.3 Pull Request Description

The Pull Request description shall include:

- Summary
- Changes

## 6. Error Handling Requirements

The tool shall show an error message when:

- No task description is provided
- The task description is empty or only whitespace

Example:

```txt
Error task description is required.
```

## 7. Non-Functional Requirements

v 0.1 shall be:

- Small
- Easy to run locally
- Easy to test
- Easy to extend 
- Independent from external services

The implementation should avoid unnecessary complexity.

## 8. Acceptance Criteria

v0.1 is complete when:

- The command-line tool can be executed
- A task description can be provided as input
- All required output items are printed
- Empty input is handled with an error message
- Unit tests can verify the generation logic
